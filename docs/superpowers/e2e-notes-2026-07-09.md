# End-to-end integration notes

**Date:** 2026-07-09

## What was run

A real end-to-end search against the live server (`node server.js`, port 5173):
`POST /search {origin:"Miami", dest:"Bogota", departDate:"2026-08-13"}` → polled
`GET /status?id=1` until `done` (~72s) → rendered `results.html?id=1`.

## Architecture path

**Path A** (spike GREEN). `runJob` ran, concurrently:
- `runGoogle` — spawned the vendored `lib/search.js` Google Flights scraper (real scrape, EUR → approx USD).
- `runAgentSearch` — spawned `claude --print --allowedTools=<kiwi>,<lastminute>` (least-privilege), which called the Kiwi + lastminute MCP tools.

Then `mergeOptions` merged + ranked + kept the top 5 with source diversity.

5 options, cheapest first, two sources represented (google + kiwi):

- **Booking links resolve:** the Kiwi option carries a real `kiwi.com/u/…` deep-link;
  Google options link to the pre-filled Google Flights page (as designed — the scraper
  has no per-flight deeplink).
- **Honest markers:** all Google prices show `~$` + "approx (converted)".
- **lastminute:** returned real fares (~$262+) but they were out-ranked out of the top 5.
  The ≥2-source cross-check rule is still satisfied (google + kiwi).

## Fix applied during this pass

Results page `fmtTime` humanizes ISO timestamps (Kiwi/lastminute give
`2026-08-13T06:24:00` → now shown as `6:24 AM`); Google's already-friendly
`10:00 AM` strings pass through unchanged.
